Problem solved: Insurance operations teams were manually compiling claims data from multiple sources into static spreadsheets, making live KPI monitoring impossible. This dashboard unified all claims and policy data into a single real-time view, enabling faster decisions on cost reserves and partner commissions.

📐 Key DAX Measures
-- Loss Ratio: claims cost paid as % of earned premium
Loss Ratio (%) =
DIVIDE(
    SUM(Claims[ClaimCostPaid]),
    SUM(Policies[EarnedPremium]), 0
) * 100

-- Policy MoM Change (Waterfall)
Policy MoM Change =
VAR CurrentMonth = SUM(Policies[ActivePolicies])
VAR PriorMonth =
    CALCULATE(
        SUM(Policies[ActivePolicies]),
        DATEADD(Policies[Date], -1, MONTH)
    )
RETURN CurrentMonth - PriorMonth

Problem solved: The marketing team lacked a unified view of how different user cohorts were converting across devices and channels. By combining Google Analytics session data with revenue metrics, this dashboard helped identify underperforming acquisition channels and informed budget reallocation.

📐 Key SQL Logic
-- New vs Returning user revenue contribution
SELECT
    user_type,
    COUNT(DISTINCT session_id)   AS sessions,
    SUM(transaction_revenue)       AS total_revenue,
    ROUND(
        SUM(transactions) * 1.0 /
        NULLIF(COUNT(DISTINCT session_id), 0), 4
    )                               AS conversion_rate
FROM analytics_sessions
WHERE date BETWEEN '2023-03-01' AND '2023-04-30'
GROUP BY user_type
ORDER BY total_revenue DESC;

Power BIMarketing

Product Activity Revenue Report

Problem solved: The product team lacked a clear view of how revenue from each cohort evolved after the signup date. By combining MoM revenue with cohort analysis, this dashboard helped identify which acquisition periods produced the highest long-term revenue.

📐 Key DAX Measures
-- MoM Revenue Growth
MoM Revenue % =
VAR Curr = SUM(Revenue[Amount])
VAR Prior = CALCULATE(
    SUM(Revenue[Amount]),
    DATEADD(Revenue[PurchaseDate], -1, MONTH))
RETURN DIVIDE(Curr - Prior, Prior, 0) * 100

-- YoY Revenue Change
YoY Revenue % =
DIVIDE(
    SUM(Revenue[Amount]) -
    CALCULATE(SUM(Revenue[Amount]),
        SAMEPERIODLASTYEAR(Revenue[PurchaseDate])),
    CALCULATE(SUM(Revenue[Amount]),
        SAMEPERIODLASTYEAR(Revenue[PurchaseDate])), 0
) * 100
Power BIMarketing

Marketing Lead Source

Problem solved: The demand generation team had no unified view of how leads progressed from creation through MQL to SQL. This dashboard connected CRM and marketing automation data to give leadership a live funnel view with historical trend analysis.

📐 Key DAX Measures
-- MQL Conversion Rate
MQL Conversion Rate =
DIVIDE(
    COUNTROWS(FILTER(Leads, Leads[Stage] = "MQL")),

-- SQL Conversion Rate (from MQL)
SQL Conversion Rate =
DIVIDE(
    COUNTROWS(FILTER(Leads, Leads[Stage] = "SQL")),
    COUNTROWS(FILTER(Leads, Leads[Stage] = "MQL")),
    0) * 100

Power BIIoT

KPI Differences

Problem solved: Quality engineers needed to quickly identify which devices had the largest deviation between two performance parameters. This dashboard ranked all devices by differential and flagged outliers, replacing a manual spreadsheet review process.

📐 Key DAX Measures
-- SQF Difference between system A and B
SQF Diff = SUM(Devices[SQF_A]) - SUM(Devices[SQF_B])

-- Flag devices outside acceptable range
Out of Range Flag =
IF(
    ABS([SQF Diff]) > 10 || ABS([EsNo Diff]) > 5,
    "Out of Range", "Within Range"
)

-- Rank by highest SQF Diff
Highest SQF Diff =
RANKX(
    ALL(Devices[SerialNumber]),
    [SQF Diff], , DESC, DENSE
)
